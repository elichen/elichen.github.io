# Coding vs Engineering

AI is making the cost of coding drop rapidly toward zero, and the effort required approaching nothing. This shift is happening faster than most people realize. Looking back over my last two years, coding a software project has compressed from two weeks to two days to two hours, driven by AI's increasing coding skill and its expanding agentic capabilities for long-horizon tasks.

This shift makes the engineer's judgment matter more.

Coding has traditionally required deep fluency in multiple layers of technical knowledge. You need syntax and language mastery, understanding programming languages at a fundamental level through textbooks, exercises, and countless hours of practice. You need ecosystem expertise across libraries, frameworks, and language-specific tools. Productive engineers needed "muscle memory" across these systems because referencing documentation mid-build was ineffective for production code. You need to learn not just what works, but what works well: gotchas, anti-patterns, and patterns that prevent future problems. And you need cultural knowledge: coding idioms, internal style guides, and community conventions. Code doesn't exist in isolation; it must pass review by other engineers through processes like pull requests.

This learning curve is substantial. At companies like Twitter, engineers typically spent six months ramping up, learning Scala, understanding distributed systems infrastructure, and internalizing internal style guides before becoming productive.

Beyond coding lies software engineering: making the decisions that determine a system's characteristics. These decisions shape performance, quality, security, and scalability in ways that either enable or constrain everything built on top. Hash map versus linked list. Schema versus unstructured data. SQL versus NoSQL. Strict event ordering versus unordered processing. SSO versus magic link authentication. Frontend framework selection. Mutable versus immutable paradigms.

These choices have consequences that spread. Selecting the wrong database leads to scaling failures, causing outages when systems can't handle growth or requiring expensive infrastructure to compensate. Poor authentication decisions create security vulnerabilities or block enterprise sales due to compliance issues.

Good engineering decisions create user value. Apple's obsessive attention to performance engineering delivers the high frame rates that make their devices feel responsive. These aren't coding problems; they're architectural and systems thinking problems.

Traditionally, software engineering education and careers have bundled coding and engineering together. Computer science programs teach algorithms alongside syntax, and job descriptions blur the lines between implementation and design. This made sense when coding itself required such significant human investment that the two skills developed together naturally.

But AI is severing the link between coding and engineering. As coding becomes free, bundling is no longer required.

This unbundling makes software engineers more valuable. When coding friction disappears, engineers can dedicate more attention to the decisions that matter: those that determine whether systems scale gracefully, whether security holds under pressure, whether the product is usable.

Engineering is a craft with deep historical roots in human problem-solving, tracing back millennia to ancient achievements like Egyptian pyramids and Roman aqueducts. In contrast, computer science emerged as an academic discipline only in the 1950s-60s, making it roughly 70 years old. The bundling of coding skills with engineering judgment happened during computer science's rapid formation, when both were necessary. But this historical pairing was circumstantial.

Engineering is about understanding available tools, recognizing constraints, making tradeoffs, and pushing the boundaries of what's possible. These decisions require something that remains uniquely human: empathy. Understanding user needs, anticipating system failures, and balancing competing priorities all draw on human experience.

Engineering also drives the research that creates better tools. Building systems and improving the tools that make them possible feed each other, and that requires human insight.

This is why coding is becoming free, but software engineering is becoming more important. As AI handles implementation, the premium on judgment, systems thinking, and empathy increases. The engineers who recognize this shift and build these skills will be more valuable.

Engineering means solving hard problems and deciding what systems should do for people. The craft remains human. AI writes the code, but humans must still decide what to build and how it should work.

Humans will always have the advantage in empathy when we build products for people, and as coding gets cheaper, that empathy becomes irreplaceable.
