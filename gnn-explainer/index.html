How a graph neural network sees a node.

A GNN doesn't classify a node from its own features alone. It moves signal across edges, compresses each neighborhood into a new hidden state, and expands what a node can know — one hop at a time.

In a node-wise MLP, every example is judged in isolation. In a GNN, a node can borrow evidence from its neighbors. That makes local structure part of the representation itself.

This piece keeps the math intentionally small: one scalar channel, weighted directed edges, and a few graph motifs that isolate message passing, receptive fields, and oversmoothing.

§ 01

The core intuition

A node never receives the whole graph in one step. Each layer reads incoming messages, aggregates them, and writes the result back into the node state. Stacking layers repeats the same local rule — which is why depth grows the receptive field.

Figure 1 traces one update in slow motion. Figure 2 fixes the graph and changes depth, so you can see when extra context helps and when repeated averaging begins to erase structure.

this simplification

Real GNNs propagate vectors, learn richer message functions, and often use residual connections or normalization. Here the hidden state is a single scalar so the mechanics stay visible.

§ 02

Depth changes what a node can know

A 0-layer model is just a node-wise classifier: it only sees xv. One layer reaches immediate neighbors. Two layers reach neighbors-of-neighbors. Beyond that, whether more depth helps depends entirely on the graph.

In the two-hop witness motif, depth uncovers the useful evidence. In the bridge motif, extra depth gradually pulls both communities toward the same representation.

read the depth control

0 behaves like an MLP on nodes. 1 and 2 expand the receptive field. Larger values show what repeated mixing does to the whole graph.

§ 03

Where GNNs help — and where they don't

GNNs help when a node's label genuinely lives in the neighborhood: homophily, motifs, bridges, bottlenecks, or evidence that sits a few hops away. They struggle when the graph is noisy, when locality is the wrong inductive bias, or when too much depth makes node states collapse together.

MLP

Depth 0

Only sees the local feature. Great when the node already contains the answer.

1L

One layer

Reads one-hop context. Useful when nearby neighbors already contain the needed evidence.

2L

Two layers

Reaches motifs and relays. Two layers is usually the right call for small-graph tasks.

∞

Too much depth

Repeated averaging flattens the graph. Distinctions shrink even when the answer stays the same.

§ takeaway

The right mental model

A GNN is not magic global reasoning. It's a local update rule applied repeatedly to a graph. Every extra layer trades more context for more mixing. The practical question is not “should I use a GNN?” but “does my node label actually live in the neighborhood — and if so, how many hops away?”
